# 命令行接口 (CLI)

`@rspack/cli` 提供了 `serve` 和 `build` 两个常用的子命令来简化日常工作。如果你尚未安装 `@rspack/cli`，请先阅读[快速开始](/guide/start/quick-start)章节。

:::warning 警告
`@rspack/cli` 旨在提供更好的使用体验，并未采用 `webpack-cli` 的设计。因此在使用上，`@rspack/cli` 和 `webpack-cli` 存在较大差异。
:::

## 开发

执行 `rspack serve` 命令，会在本地启动一个开发服务器，监听文件变化并自动刷新页面。

```sh
$ rspack serve
$ rspack serve -c ./your.config.js
```

## 构建

执行 `rspack build` 命令，将会在 dist 目录下构建生产版本。

```sh
$ rspack build  # 默认读取 rspack.config.js 配置文件
$ rspack build -c ./your.config.js
```

## 选项

Rspack 支持以下命令行选项，你可以通过 `--help` 查看所有的命令行选项。

| 选项          | 描述                       | 类型    | 默认值 |
| ------------- | -------------------------- | ------- | ------ |
| -c, --config  | 配置文件路径               | string  |        |
| --entry       | 入口文件路径               | array   |        |
| --output-path | 指定输出文件目录的绝对路径 | string  |        |
| -m, --mode    | 构建模式                   | string  |        |
| -w, --watch   | 监听文件变化               | boolean | false  |
| --devtool     | 控制是否生成 sourcemap     | boolean | false  |
| --analyze     | 是否开启构建分析           | boolean | false  |
| --json        | 输出打包信息为 JSON 文件   |         |        |
| -h, --help    | 显示帮助信息               | boolean |        |
| -v, --version | 显示版本号                 | boolean |        |
