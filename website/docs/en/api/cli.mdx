# Command Line Interface (CLI)

`@rspack/cli` provides two common subcommands, serve and build, to simplify daily work. If you do not have `@rspack/cli` installed, please read the [Quick Start](/guide/start/quick-start) section first.

:::warning
`@rspack/cli` is designed to offer a better user experience and does not adopt the design of `webpack-cli`. As a result, there are significant differences in usage between `@rspack/cli` and `webpack-cli`.
:::

## Serve

Running the `rspack serve` command will start a local development server that listens for file changes and automatically refreshes the page.

```sh
$ rspack serve
$ rspack serve -c ./your.config.js
```

## Build

Running the `rspack build` command will build the production version in the dist directory.

```sh
$ rspack build # Read the rspack.config.js configuration file by default
$ rspack build -c ./your.config.js
```

## Options

Rspack supports the following command-line options, you can see all of them with `--help`.

| Options       | Description                               | Type    | Default |
| ------------- | ----------------------------------------- | ------- | ------- |
| -c, --config  | Path to configuration file                | string  |         |
| --entry       | Entry file path(s)                        | array   |         |
| --output-path | Absolute path to specify output directory | string  |         |
| -m, --mode    | Build mode                                | string  |         |
| -w, --watch   | Watch for file changes                    | boolean | false   |
| --devtool     | Control if sourcemaps are generated       | boolean | false   |
| --analyze     | Enable build analysis                     | boolean | false   |
| --json        | Output bundle information as JSON file    |         |         |
| -h, --help    | Display help information                  | boolean |         |
| -v, --version | Display version number                    | boolean |         |
